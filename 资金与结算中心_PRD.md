## 二、功能定义

### 2.1 功能名称

资金与结算中心

### 2.2 功能定位

资金与结算中心用于统一展示企业的经营账、结算账、资金账，帮助财务、老板、运营负责人快速判断：

- 本月经营收入、费用、税费是多少
- 亚马逊交易中已发放、延迟发放金额是多少
- 结算账期应收多少、已转账多少
- 第三方收款账户实际到账多少
- 经营账、结算账、资金账差异来自哪里

本功能不再追求不同官方报表之间的绝对平账，而是围绕差异识别、差异解释、资金追踪建立统一入口。

### 2.3 菜单位置

```text
财务
  资金账户
    资金与结算中心
```

原“回款对账”能力合并到本中心，作为资金到账核对与差异核查的一部分。

### 2.4 面向对象

- 财务：核对 Summary、付款结算、第三方收款账户到账差异
- 老板：查看经营结果、结算释放、到账进度
- 运营负责人：查看店铺维度的延迟释放、未到账风险
- 客户成功：辅助客户解释 Amazon 报表与系统数据差异

## 三、核心结论

### 3.1 本期数据源

| 数据源 | 系统来源 | 是否需要客户上传 | 用途 |
| --- | --- | --- | --- |
| Summary PDF | 客户从 Amazon 后台下载并上传 | 需要 | 经营账汇总，解析 Income / Expenses / Tax / Transfers |
| Transaction CSV | Amazon 后台可下载文件 | 不需要 | 不再作为系统导入与解析对象，仅作为人工排查参考 |
| 付款结算-交易一览 | 系统通过 Amazon Finance API 同步 | 不需要 | 按交易结算时间统计交易明细，识别 Released / Deferred，按费用类型映射为收入、支出、税费 |
| 付款结算-所有结算 | 系统通过 Amazon Finance API 同步 | 不需要 | 按结算账期统计应收金额、转账金额、转账状态 |
| 第三方收款账户到账流水 | 收款工具接口或导入 | 不需要或按现有能力导入 | 核对实际到账、手续费、汇损 |

### 3.2 CSV 处理原则

系统不再要求客户上传或解析 Transaction CSV。

客户侧统一说明：

```text
后续只需要上传 Summary PDF。
交易明细、已发放、延迟发放、结算账期、转账状态由系统付款结算数据提供。
Transaction CSV 仅作为 Amazon 后台人工下载的参考文件，不作为日常对账导入资料。
```

产品与研发侧原则：

- 不设计 CSV 导入入口
- 不维护 CSV 多语言表头解析
- 不用 CSV 作为主数据
- 若客户提供 CSV，只用于人工排查系统接口数据是否缺失

### 3.3 时间口径

| 账层 | 统计口径 | 说明 |
| --- | --- | --- |
| 经营账 | Summary PDF 报告区间 | 解析 PDF 中 Account activity 对应月份的 Income / Expenses / Tax |
| 交易明细 | 付款结算-交易一览的结算时间 | 用于替代 Transaction CSV，统计本月交易入账、Released / Deferred 与费用科目 |
| 结算账期 | 付款结算-所有结算的结算关闭时间 | 用于统计本月关闭的账期、应收金额、转账状态 |
| 资金账 | 所有结算转账时间 + 第三方收款账户到账时间 | 用于核对平台已转账金额与实际到账金额 |

关键规则：

```text
交易时间、账期关闭时间、转账时间、到账时间是四个不同时间。
不同时间口径产生跨月差异属于正常现象，系统需要识别并解释，而不是强行平账。
```

### 3.4 真实样本验证结论

基于 A51-UK 2026 年 5 月样本：

- Summary PDF 不包含 Transaction Status
- Summary PDF 不包含 Transaction Release Date
- Summary PDF 不包含 SettlementID
- Transaction CSV 的 date/time 为 posted date，不是订单下单时间，也不是账期关闭时间
- 同一月份的 Transaction CSV 中，会出现结算关闭月份为下月的 SettlementID
- 付款结算-所有结算中，一个结算账期只有一个唯一 SettlementID
- 交易一览中，一个 SettlementID 对应多条交易明细
- 按结算关闭时间筛选 5 月账期得到的 SettlementID，可以在交易一览中查询对应明细
- 交易一览按结算时间筛选 5 月时，可能包含 6 月关闭账期的 SettlementID，这属于跨月结算
- 账期关闭后，SettlementID、追踪编号、最终转账信息可能存在延迟回填
- SettlementID 为空但系统结算编号存在时，账期进入“待回填”状态，不能判定为异常或未到账
- 用 CSV 中出现过的 settlement id 反查交易一览完整账期，会导致查询范围膨胀，不能作为 PDF 月度核对口径

样本关键数据：

| 指标 | 金额 |
| --- | ---: |
| PDF Income | GBP 60,521.79 |
| PDF Expenses | GBP -36,295.82 |
| PDF Tax | GBP 759.20 |
| PDF Transfers | GBP -22,660.23 |
| PDF 经营净额，不含转账 | GBP 24,985.17 |
| PDF 含转账后净变化 | GBP 2,324.94 |
| Transaction CSV 合计 | GBP 2,324.94 |
| 所有结算 5 月关闭账期应收合计 | GBP 22,660.23 |
| 所有结算 5 月关闭账期转账合计 | GBP 22,660.23 |
| 交易一览 Deferred 金额 | GBP 767.78 |

结论：

```text
Summary PDF 与 Transaction CSV 是 Amazon 报告区间口径。
付款结算-所有结算是账期关闭口径。
付款结算-交易一览可以覆盖本功能所需的 Transaction CSV 核心字段，因此系统无需解析 CSV。
```

## 四、业务模型

### 4.1 经营账

| 项 | 内容 |
| --- | --- |
| 数据来源 | Amazon Summary PDF |
| 统计口径 | PDF 报告区间 |
| 核心字段 | Income、Expenses、Tax、Transfers |
| 核心用途 | 税务报表、经营分析、经营账核对 |
| 注意事项 | PDF 只有汇总结果，没有交易状态、释放日期、SettlementID |

经营净额：

```text
经营净额 = Income + Expenses + Tax
```

PDF 中 Transfers 单独展示，用于参考 Amazon 报告内的转账变化，不参与经营净额。

### 4.2 交易明细账

| 项 | 内容 |
| --- | --- |
| 数据来源 | 付款结算-交易一览 |
| 统计口径 | 交易一览的结算时间 |
| 核心字段 | 结算时间、交易状态、交易类型、费用类型、费用明细、金额、币种、SettlementID |
| 核心用途 | 替代 CSV 明细核对，统计 Released / Deferred，解释经营账差异 |

交易明细账用于回答：

- 本月交易明细中已发放多少
- 本月交易明细中延迟发放多少
- 本月交易明细按费用类型可归为多少收入、支出、税费
- 本月交易涉及哪些 SettlementID
- 哪些 5 月交易进入了 6 月关闭账期

### 4.3 结算账期账

| 项 | 内容 |
| --- | --- |
| 数据来源 | 付款结算-所有结算 |
| 统计口径 | 结算关闭时间 |
| 核心字段 | SettlementID、结算开始时间、结算关闭时间、应收金额、转账金额、转账状态、转账时间、银行尾号、追踪编号 |
| 核心用途 | 应收账款、转账状态、账期维度对账 |

账期金额口径：

```text
一个 SettlementID = 一个结算账期。
账期应收金额 = 该结算账期整体应收结果。
```

注意：

```text
账期应收金额不能直接代表某个月交易筛选结果。
同一 SettlementID 内可能包含不同 posted date 或不同业务时间的交易明细。
账期关闭不代表 SettlementID、追踪编号、转账信息已经全部回填。
```

账期标识：

| 标识 | 来源 | 用途 |
| --- | --- | --- |
| 系统结算编号 | 系统付款结算内部编号 | 账期创建后即可存在，用于系统内部临时关联 |
| SettlementID | Amazon 官方结算编号 | 回填后用于精确关联交易明细、资金到账、外部对账 |
| 追踪编号 | Amazon 转账追踪信息 | 回填后用于与第三方收款账户或银行流水匹配 |

当 `结算状态 = 关闭` 且 `SettlementID = 空` 时，系统不判定为异常，状态展示为：

```text
待回填 SettlementID
```

### 4.4 资金账

| 项 | 内容 |
| --- | --- |
| 数据来源 | 付款结算-所有结算 + 第三方收款账户到账流水 |
| 统计口径 | Amazon 转账时间、第三方收款账户到账时间 |
| 核心字段 | 店铺、币种、转账金额、到账金额、手续费、汇损、银行尾号、追踪编号 |
| 核心用途 | 到账核对、未到账预警、现金流管理 |

资金账用于回答：

- Amazon 是否已经转账
- 第三方收款账户是否已经到账
- 到账金额与转账金额是否一致
- 差异来自手续费、汇损、在途，还是匹配失败

## 五、核对逻辑

### 5.1 经营账核对

查询逻辑：

```text
1. 用户上传目标月份 Summary PDF
2. 系统解析 PDF Income / Expenses / Tax / Transfers
3. 系统查询付款结算-交易一览中结算时间落在目标月份的交易
4. 按费用映射规则将交易一览金额归为收入、支出、税费、转账、预留金
5. 使用 PDF Income / Expenses / Tax 与交易一览归类结果核对
```

核对结果：

```text
经营账差异 = PDF 经营净额 - 交易一览经营净额
```

经营账差异原因优先级：

- 费用类型映射未覆盖
- 时区边界差异
- 跨月结算
- Amazon 报告区间与接口交易时间口径差异
- 接口同步缺失或分页异常

### 5.2 Released / Deferred 统计

查询逻辑：

```text
付款结算-交易一览
where 结算时间 in 目标月份
group by 交易状态
```

状态：

| 状态 | 含义 |
| --- | --- |
| RELEASED | 已发放 |
| DEFERRED | 延迟发放 |
| DEFERRED_RELEASED | 延迟后已发放，如接口存在该状态，需避免与 RELEASED 重复计入 |

展示指标：

- 已发放金额
- 延迟发放金额
- 延迟交易笔数
- 涉及店铺数
- 涉及 SettlementID 数

### 5.3 结算账期对平

账期详情对平逻辑：

```text
1. 在付款结算-所有结算中选择一个账期
2. 若 SettlementID 已回填，用 SettlementID 查询付款结算-交易一览明细
3. 若 SettlementID 未回填，使用系统结算编号临时查询或展示账期，等待后续同步回填
4. 查询时可放宽时间范围，时间仅作为检索范围
5. 归属判断优先以 SettlementID 为准；SettlementID 未回填时以系统结算编号临时关联
6. 排除 Transfer 转账行后，汇总交易明细金额
7. 与所有结算中的应收金额核对
```

重要规则：

```text
账期对平优先以 SettlementID 为主键。
SettlementID 未回填时，不参与 SettlementID 精确对账。
不要用交易一览的结算时间判断明细是否属于该账期。
不要用同一个 SettlementID 下的 Transfer 行核对账期应收金额。
```

原因：

- Transfer 行是资金转账动作，不是经营交易明细
- Amazon Transaction 报告中的 Transfer 行可能挂在下一个 settlement id 下
- 所有结算中的转账金额才是资金账锚点
- 账期关闭后，Amazon 官方 SettlementID 与追踪编号可能延迟回填

### 5.4 CSV settlement id 使用边界

若客户提供 Transaction CSV，CSV 中的 settlement id 仅用于人工定位与差异解释，不能作为系统月度核对主查询条件。

错误做法：

```text
1. 先取 5 月 CSV 中出现过的 settlement id
2. 再用这些 settlement id 去交易一览查询完整明细
3. 将查询结果与 5 月 PDF 或 5 月 CSV 汇总核对
```

原因：

```text
CSV 5 月数据 = date/time 落在 5 月的交易截面。
SettlementID 完整明细 = 该账期下全部交易，可能覆盖 4 月、5 月、6 月。
CSV 5 月交易只是这些完整账期交易的一个子集。
```

正确做法：

```text
PDF 月度核对：交易一览按结算时间筛选目标月份。
账期完整对平：所有结算按结算关闭时间筛选，再用 SettlementID 查询完整交易明细。
```

### 5.5 跨月结算识别

当交易一览按目标月份查询后，关联到的 SettlementID 关闭月份不是目标月份，则识别为跨月结算。

示例：

```text
交易结算时间：2026-05
SettlementID：27214060952
账期关闭时间：2026-06
```

系统解释：

```text
该部分交易属于 5 月经营交易，但进入 6 月关闭账期，因此会造成 5 月经营账与 5 月结算账期金额不一致。
```

### 5.6 SettlementID 待回填处理

当所有结算中出现以下情况：

```text
结算状态 = 关闭
SettlementID = 空
转账状态 = 转账中 或 已转账
追踪编号 = 空
```

系统处理规则：

- 状态展示为“待回填 SettlementID”或“转账中”
- 不判定为未到账
- 不参与 SettlementID 精确匹配
- 可用系统结算编号展示账期记录和临时交易明细
- 下一次同步回填 SettlementID 后，自动重新核算
- 若超过配置时长仍未回填，进入“结算信息回填异常”

建议阈值：

| 状态 | 阈值 | 处理 |
| --- | --- | --- |
| 账期关闭后 SettlementID 未回填 | 24 小时内 | 正常等待 |
| 账期关闭后 SettlementID 未回填 | 超过 24 小时 | 提示待回填 |
| 账期关闭后 SettlementID 未回填 | 超过 72 小时 | 标记为回填异常 |

### 5.7 资金到账核对

查询逻辑：

```text
1. 在付款结算-所有结算中按结算关闭时间筛选目标月份账期
2. 取已转账状态的转账金额、转账时间、银行尾号、追踪编号
3. 与第三方收款账户到账流水匹配
4. 计算到账金额、手续费、汇损、未到账金额
```

匹配优先级：

| 优先级 | 匹配条件 |
| --- | --- |
| P0 | 追踪编号完全一致 |
| P1 | SettlementID / 外部业务号一致 |
| P2 | 系统结算编号一致，且 SettlementID 暂未回填 |
| P3 | 店铺 + 币种 + 金额 + 银行尾号一致 |
| P4 | 店铺 + 币种 + 金额 + 转账时间后合理到账窗口 |

到账状态：

| 状态 | 判定 |
| --- | --- |
| 已到账 | 第三方收款账户到账金额已匹配，且金额差异在阈值内 |
| 部分到账 | 已匹配到账流水，但到账金额小于应到账金额 |
| 未到账 | 已转账但未匹配到第三方收款账户到账 |
| 转账中 | 所有结算显示转账中，或 SettlementID / 追踪编号尚未回填 |
| 待回填 | 账期已关闭，但 SettlementID 为空 |
| 匹配异常 | 金额、币种、店铺或银行尾号不一致 |

## 六、费用类型映射

### 6.1 标准科目

交易一览中的费用类型统一映射为以下标准科目：

| 标准科目 | 说明 |
| --- | --- |
| Income | 商品销售、运费收入、礼品包装收入、促销折扣、补偿、清算收入等 |
| Expenses | 平台佣金、FBA 费用、广告费、服务费、数字服务费、库存费、其他平台费用等 |
| Tax | 商品税、运费税、平台代扣税、税费退款等 |
| Transfers | Amazon 转账行，仅用于资金参考，不参与经营净额核对 |
| Reserve | 预留金、预留释放等内部资金流转，单独列示或净额处理 |
| Unmapped | 未识别费用类型，进入待映射清单 |

### 6.2 初始映射规则

| 交易一览字段 | 标准科目 | 处理说明 |
| --- | --- | --- |
| ProductCharges / OurPricePrincipal | Income | 商品销售收入 |
| Shipping / ShippingPrincipal | Income | 运费收入 |
| PromoRebates | Income | 促销折扣，通常为负数 |
| RecommerceLiquidation | Income | 清算收入或清算调整 |
| FBAInventoryReimbursement | Income | FBA 库存补偿 |
| Other / GiftwrapPrincipal | Income | 礼品包装等其他收入 |
| AmazonFees | Expenses | 平台佣金、配送费、扣费等 |
| FBAFees | Expenses | 仓储、长期仓储、退货处理等 |
| DigitalServicesFee | Expenses | 数字服务费 |
| ProductAdsPayment / Base | Expenses | 广告扣费 |
| ServiceFee | Expenses | 服务类费用 |
| Tax | Tax | 税费、平台代扣税、税费退款 |
| Transfer / FundTransfer | Transfers | 资金转账，不参与经营净额 |
| ReserveCredit / ReserveDebit | Reserve | 预留金流转，单独列示 |

### 6.3 未映射费用处理

若交易一览出现未识别费用类型：

- 不直接归入收入、支出、税费
- 标记为 Unmapped
- 在详情页展示费用类型、费用明细、金额、笔数
- 允许产品或财务维护映射规则
- 重新核算后更新历史结果

### 7.1.1 首页看板数据来源与计算公式

#### 7.1.1.1 统一查询条件

首页看板按筛选条件查询：

| 参数 | 说明 |
| --- | --- |
| platform | 平台，当前为 Amazon |
| month | 对账月份，例如 2026-05 |
| store_id | 店铺，可为空；为空时查询当前权限下全部店铺 |
| currency | 看板固定 USD 折算；列表与详情使用原币种 |
| month_start | 月份开始时间，含当天 00:00:00 |
| month_end | 下月开始时间，不含当天 00:00:00 |

跨币种折算规则：

```text
USD金额 = 原币种金额 * 月末汇率(currency -> USD)
看板所有跨店铺金额必须先折算 USD 后再汇总。
列表和详情不做跨币种汇总，保留原币种。
```

#### 7.1.1.2 基础数据集

| 数据集 | 来源 | 查询条件 | 用途 |
| --- | --- | --- | --- |
| S_summary | Summary PDF 解析结果 | store_id + report_month = month | 经营账收入、支出、税费、经营净额 |
| T_transaction | 付款结算-交易一览 | store_id + settlement_time >= month_start + settlement_time < month_end | 交易净额、Released / Deferred、费用科目归类、跨月 SettlementID |
| A_settlement | 付款结算-所有结算 | store_id + settlement_close_time >= month_start + settlement_close_time < month_end | 账期应收、已转账、待转账、账期状态 |
| R_receipt | 第三方收款账户到账流水 | 与 A_settlement 按店铺、币种、追踪编号、银行尾号、金额区间匹配 | 已到账、未到账、到账率、手续费、汇损 |

注意：

```text
T_transaction 按结算时间查询，用于经营账核对与释放状态统计。
A_settlement 按结算关闭时间查询，用于账期应收与转账状态统计。
R_receipt 按匹配结果进入资金账，不直接按到账时间替代账期金额。
```

#### 7.1.1.3 顶部指标公式

| 指标 | 数据来源 | 计算公式 |
| --- | --- | --- |
| 经营收入 | S_summary | SUM_USD(pdf_income) |
| 经营支出 | S_summary | SUM_USD(pdf_expenses)，支出按负数存储，页面可展示绝对值或负数 |
| 经营税费 | S_summary | SUM_USD(pdf_tax) |
| 经营净额 | S_summary | SUM_USD(pdf_income + pdf_expenses + pdf_tax) |
| 交易净额 | T_transaction | SUM_USD(mapped_income + mapped_expenses + mapped_tax)，Transfer 不参与经营净额 |
| 已发放金额 | T_transaction | SUM_USD(amount) WHERE transaction_status = Released AND standard_subject IN Income / Expenses / Tax |
| 延迟发放金额 | T_transaction | ABS(SUM_USD(amount)) WHERE transaction_status = Deferred |
| 延迟笔数 | T_transaction | COUNT(transaction_id) WHERE transaction_status = Deferred |
| 释放率 | T_transaction | 已发放金额 / (已发放金额 + 延迟发放金额) |
| 本月关闭账期数 | A_settlement | COUNT(DISTINCT COALESCE(settlement_id, system_settlement_no)) |
| 账期应收金额 | A_settlement | SUM_USD(receivable_amount) |
| 已转账金额 | A_settlement | SUM_USD(transfer_amount) WHERE transfer_status = 已转账 |
| 待转账金额 | A_settlement | 账期应收金额 - 已转账金额 |
| 已到账金额 | R_receipt | SUM_USD(receipt_amount) WHERE receipt_status = 已匹配 |
| 未到账金额 | A_settlement + R_receipt | 已转账金额 - 已到账金额 |
| 到账率 | A_settlement + R_receipt | 已到账金额 / 已转账金额 |
| 异常店铺数 | 对账结果表 | COUNT(DISTINCT store_id) WHERE 对账状态 IN 差异 / 未到账 / 部分到账 / 待回填 |

#### 7.1.1.4 首页组件公式

| 组件 | 查询/计算逻辑 |
| --- | --- |
| 店铺到账进度排行 | 按 store_id 聚合：到账率 = SUM(receipt_amount_matched) / SUM(transfer_amount_transferred)，按到账率升序或未到账金额降序展示 |
| 三账趋势 | 按近 6 个月分别计算：经营净额 = S_summary 经营净额；结算净额 = T_transaction 交易净额；到账金额 = R_receipt 已匹配到账金额 |
| 资金链路 | 当前筛选月份展示三段金额：经营账 = S_summary 经营净额；结算账 = A_settlement 已转账金额，辅助展示账期应收金额；资金账 = R_receipt 已到账金额 |
| 差异预警排行 | 按店铺计算风险金额 = ABS(经营差异) + 延迟发放金额 + 未到账金额；按风险金额降序展示 |
| 差异说明 | 根据差异规则生成：跨月结算、延迟发放、待回填 SettlementID、到账在途、已转账未匹配流水、未映射费用 |

#### 7.1.1.5 差异字段公式

| 差异字段 | 公式 |
| --- | --- |
| 经营差异 | S_summary 经营净额 - T_transaction 交易净额 |
| 账期差异 | T_transaction 已发放金额 - A_settlement 账期应收金额 |
| 到账差异 | A_settlement 已转账金额 - R_receipt 已到账金额 |
| 差异率 | ABS(经营差异) / ABS(S_summary 经营净额) |
| 跨月 SettlementID 数 | COUNT(DISTINCT settlement_id) WHERE T_transaction.settlement_time 在本月 AND A_settlement.settlement_close_time 不在本月 |

## 九、差异规则

### 9.1 经营差异

| 状态 | 判定 |
| --- | --- |
| 平账 | 差异金额 = 0 |
| 轻微差异 | 0 < 绝对差异 < 1 USD 等值 |
| 平台差异 | 绝对差异 >= 1 USD 等值 |
| 待映射 | 存在 Unmapped 费用类型 |
| 数据缺失 | PDF 未导入或交易一览数据缺失 |

### 9.2 释放状态

| 状态 | 判定 |
| --- | --- |
| 全部已发放 | Deferred 金额 = 0 |
| 存在延迟发放 | Deferred 金额 > 0 |
| 延迟超期 | Deferred 超过配置天数仍未释放 |

### 9.3 到账状态

| 状态 | 判定 |
| --- | --- |
| 已到账 | 已转账金额与第三方收款账户到账金额匹配 |
| 部分到账 | 到账金额小于应到账金额 |
| 未到账 | 所有结算显示已转账，但未匹配到账 |
| 转账中 | 所有结算转账状态为转账中，或处于合理到账窗口内 |
| 待回填 | 账期已关闭，但 SettlementID 或追踪编号尚未回填 |
| 在途 | Amazon 已转账，仍在合理到账窗口内 |
| 匹配异常 | 存在金额、币种、店铺、尾号或追踪编号不一致 |

### 9.4 自动差异说明

系统按优先级生成差异说明：

1. PDF 未导入或解析失败
2. 交易一览存在未映射费用类型
3. 交易结算时间与账期关闭时间跨月
4. 存在 Deferred 延迟发放
5. 账期已关闭但 SettlementID 或追踪编号待回填
6. 所有结算已转账但第三方收款账户未到账
7. 到账金额存在手续费或汇损差异
8. 接口同步数据缺失或分页异常

示例说明：

```text
A51-UK 2026-05 经营账与结算账存在 GBP 582.39 差异。
主要原因：部分 5 月交易进入 6 月关闭账期，同时存在 GBP 767.78 延迟发放金额。
系统已识别 7 个跨月 SettlementID，可在结算账期 Tab 查看明细。
```

## 十四、研发实现要点

1. Summary PDF 解析只负责经营账汇总，不承担释放状态与 SettlementID 解析。
2. 交易一览按结算时间查询，用于替代 CSV 的核心明细能力。
3. 所有结算按结算关闭时间查询，用于账期和资金口径。
4. 根据交易一览 SettlementID 可以回查所有结算，展示账期归属和转账状态；SettlementID 未回填时，用系统结算编号临时展示。
5. 按账期对平时，优先以 SettlementID 查询完整交易明细；SettlementID 为空时不参与精确对账，等待后续同步回填。
6. 账期应收对平需排除 Transfer 行。
7. 资金到账核对以所有结算的转账金额为锚点，不以 Transaction 报告中的 Transfer 行为锚点。
8. 费用类型映射必须配置化，未映射费用进入待处理清单。
9. 历史核算结果需要保留快照，避免 API 状态回刷导致历史页面变化。
10. 首页到账进度按店铺统计，不按国家站点统计。
11. CSV 中的 settlement id 仅用于人工定位，不允许作为月度核对的主查询条件。
12. 账期关闭但 SettlementID 为空时，状态为“待回填”，不能直接判定为异常或未到账。
13. 跨店铺汇总看板必须按 USD 统一折算；原币种只用于列表、详情和单店铺核对，不允许跨币种直接加总。
